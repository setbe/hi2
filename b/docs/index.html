Documentation for b/ and public b:: API

Keywords: capabilities, os middleware, thin wrappers, no OS types in public API, RAII for ownership resources.

b:: is a capability layer between pure a:: and higher frameworks in c::. It is intentionally thin: it exposes stable API contracts and delegates platform specifics to port backends.

Dependency Law

  • b may depend only on a.
  • Public b/*.hpp headers must stay platform-neutral.
  • Platform backends live in b/ports/<platform>/*.cpp.
  • c:: depends on capabilities, not on OS names.

Error Model

Socket receive no longer uses magic values like -1/-2. It returns socket_recv_result { status, bytes, from }. 

b::socket_recv_result r = b::socket_recv_from(sock, out);
if (r.status == b::socket_status::ok) {
    // r.bytes bytes from r.from
} else if (r.status == b::socket_status::would_block) {
    // non-blocking socket has no packet yet
}

RAII Policy in b::

  • Use RAII for ownership resources (socket/file/window/map).
  • Use free functions for stateless queries (time, yield, current id).
  • Do not grow b into high-level protocol/framework layer.
