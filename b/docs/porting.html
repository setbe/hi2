Porting b:: to a Custom OS

Implement only the capabilities your environment supports.

Steps

  1. Set capability macros in build config (B_HAS_TIME, B_HAS_THREADS, B_HAS_SOCKETS).
  2. Add new backend files under b/ports/<your_os>/.
  3. Implement functions from b/detail/port_contract.hpp for enabled capabilities.
  4. Wire backend sources in CMake for your target platform.

Rule of Thumb

b is thin middleware. If an API becomes protocol/business logic, it belongs to c::, not b::.
