Mental model: memory_req describes, layout composes, typed_scope executes.
Plain example: “I need 3 typed components” -> calculate bytes/alignment at compile-time -> bind those slots to one runtime arena frame.

typed_scope Invariants

Usability

active() / operator bool() defines whether a scope can be used. init_status() stores initialization result only.

Cleanup vs Close

destroy_all() cleans constructed objects only. release() closes the frame and makes the scope inert.

Type Safety

slot/get/construct/destroy statically verify that T belongs to the Layout.

Nothrow Policy

For typed construction, components must be nothrow-constructible and nothrow-destructible.

Core invariant: if active()==false, the scope is not usable; if scope is inert, constructed-state must be empty.

a/bounded.hppbounded, pow2_bounded, static_boundedRepresent dimensions/counts/alignment as validated types
  • require: caller/API preconditions.
  • require_that<Tag>: runtime check that mints a proof token.
  • assume_verified(proven<Tag>&&): consume one live proof token on trusted path.
  • Trusted path in bounded types is internal-only construction, not a user-input bypass.

How to Use This Atlas in Practice

  • When implementing a module: start from the tables above, then lock contracts in code.
  • When reviewing a PR: validate that every new API matches an intent from this atlas.
  • When debugging: map failing behavior to violated contract category first (precondition, invariant, lifetime).
