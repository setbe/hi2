Required C++ Knowledge for Starting with Bare-Metal

Цей розділ написаний як технічний вступ-підручник, а не як короткий конспект. Ми рухаємось від знайомих C++ понять до глибших принципів, на яких тримається a::.

Якщо термінів багато, читай по одному “iceberg-рівню” за сесію і проганяй приклади вручну. Мета — побудувати інтуїцію, а потім формальну дисципліну.

Iceberg Рівень 1: Компіляція, Лінкування, Виконання

Для цієї архітектури compile-time — частина коректності, а не декоративна оптимізація. Багато рішень свідомо переноситься в compile-time, щоб runtime-поведінка була передбачуваною. 

constexpr a::u32 kib(a::u32 n) noexcept { return n * 1024u; }

constexpr a::u32 boot_pool = kib(64); // compile-time

a::u32 pages = 3;
a::u32 runtime_pool = kib(pages); // runtime

Ключ: constexpr означає “може бути compile-time”, а не “завжди compile-time”.

Iceberg Рівень 2: Lifetime та Ownership

У a:: пам'ять і ресурси мають явний життєвий цикл. RAII тут не “стиль”, а механізм стабільності. 

void phase(a::arena_view& arena) {
    a::scope frame(arena); // mark
    // temporary work
} // rewind на деструкторі

Iceberg Рівень 3: Вказівники і Розмір

Вказівник дає адресу, але не гарантує довжину і семантику буфера. Саме тому потрібні явні view-контракти. 

a::u8 raw[256]{};
a::usize n = 128;
a::view<a::u8> dyn{raw, n};

Iceberg Рівень 4: view<T> проти view_n<T,N>

view<T> — runtime-розмір, view_n<T,N> — compile-time-розмір. Для криптографії fixed-size view часто кращий, бо розмір є частиною контракту. 

a::u8 key[32]{};
a::u8 nonce[12]{};

a::byte_view_n<32> key_v = a::as_view(key);
a::byte_view_n<12> nonce_v = a::as_view(nonce);

Iceberg Рівень 5: Padding і Alignment

Alignment — закон адреси, padding — наслідок цього закону у layout. Ігнорування будь-якого пункту ламає memory-математику. 

struct packet_header {
    a::u8  type;
    a::u32 size;
};

// типово:
// alignof(packet_header) == 4
// sizeof(packet_header)  == 8
start_b = align_up(a.bytes, b.align)
result.bytes = start_b + b.bytes
result.align = max(a.align, b.align)

Iceberg Рівень 6: Memory-frame і Rewind

a::u8 storage[64 * 1024]{};
a::arena_view arena{storage, sizeof(storage)};

{
    a::scope frame(arena);
    // alloc A/B/C
} // rewind

Один mark + один rewind дають детермінований cleanup тимчасової фази.

Iceberg Рівень 8: Freestanding vs Bare-Metal

Freestanding означає мінімальні припущення про runtime/стандартну бібліотеку. Bare-metal означає роботу близько до апаратного середовища, часто без повної ОС.

Тому a:: не займається отриманням RAM від ОС. Це відповідальність вищого шару (b) або користувача.

Контракт шарів: a визначає детерміновані примітиви; b дає environment capabilities; c/d будують вищі моделі зверху.

Міні-Вправи Для Самоперевірки

  3. Для parser API розділи перевірки на require і invariant.
  4. Поясни, чому hidden allocation ламає гарантії шару a.
