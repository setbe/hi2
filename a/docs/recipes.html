Practical Recipes

Copy-ready templates for common patterns in a::.

1. Compile-time peak memory across contexts

struct boot_a { static constexpr a::memory_req requirement() noexcept { return {24 * 1024, 16}; } };
struct boot_b { static constexpr a::memory_req requirement() noexcept { return {40 * 1024, 16}; } };
struct run_a  { static constexpr a::memory_req requirement() noexcept { return {80 * 1024, 16}; } };
struct run_b  { static constexpr a::memory_req requirement() noexcept { return {32 * 1024, 16}; } };

using boot_ctx = a::sum_req<boot_a, boot_b>;
using run_ctx  = a::sum_req<run_a, run_b>;
using peak_ctx = a::max_req<boot_ctx, run_ctx>;

constexpr a::memory_req app_req = peak_ctx::requirement();

This computes a deterministic memory ceiling: each context is packed with sum_req, then global peak is selected with max_req.
Plain example: boot may need 64 KB, gameplay 96 KB, so you allocate 96 KB once.

2. Typed slots with explicit construction

struct parser_state {
    static constexpr a::memory_req requirement() noexcept { return {sizeof(parser_state), alignof(parser_state)}; }
    parser_state() noexcept = default;
    ~parser_state() noexcept = default;
};

using parser_layout = a::layout<parser_state>;

void run_parser(a::arena_view arena) {
    a::typed_scope<parser_layout> frame(arena);
    if (!frame.active()) return;

    parser_state* st = frame.construct<parser_state>();
    if (!st) return;

    // work

    frame.destroy<parser_state>();
    frame.release();
}

This recipe combines explicit and RAII cleanup: destroy<T>() destroys one concrete object immediately, while release() closes the frame and makes the scope inert.
Plain example: you parse one file, call destroy<parser_state>() to end that object now, then call release() to close the whole scope; if you forget, ~typed_scope() still runs destroy_all() as a safety fallback.

3. Reusable scratch with arena scope

void decode_batch(a::arena_view arena) {
    {
        a::scope phase(arena);
        auto r = arena.alloc(48 * 1024, 16);
        if (!a::is_ok(r.code)) return;
        // decode image set A
    } // rewind to phase mark

    {
        a::scope phase(arena);
        auto r = arena.alloc(48 * 1024, 16);
        if (!a::is_ok(r.code)) return;
        // decode image set B using same bytes
    }
}

Each inner a::scope rewinds arena offset automatically, so the same scratch bytes are safely reused between phases.
Plain example: decode icon A, scope ends, then decode icon B into the same 48 KB region.

4. Two-pass module skeleton

namespace my {
struct plan { a::u32 count; };
struct input { a::u32 max_count; };
struct output { a::u32 written; };
}

namespace a::two_pass {
template<>
struct traits<my::plan, my::input> {
    static plan_result<my::plan> plan(const my::input& in) noexcept {
        if (in.max_count == 0) return {status::invalid_input, {}, {}};
        my::plan p{in.max_count};
        return {status::ok, p, {in.max_count * sizeof(a::u32), alignof(a::u32)}};
    }

    static build_result<my::output> build(const my::plan& p, a::arena_view arena) noexcept {
        auto r = arena.alloc(p.count * sizeof(a::u32), alignof(a::u32));
        if (!is_ok(r.code)) return {r.code, {0}};
        return {status::ok, {p.count}};
    }
};
}

This template shows a full two-pass contract: validate input in plan(), then allocate and produce output in build().
Plain example: first count rows and bytes for CSV, then allocate exactly that amount and write parsed rows.

5. Minimal crypto flow

a::crypto::sha256 h;
h.update(data_a);
h.update(data_b);
a::crypto::sha256::digest d = h.final();

// optionally reuse object
h.reset();

This is the minimal streaming hash pattern: update in chunks, finalize digest, optionally reset instance for reuse.
Plain example: read a file in 4 KB chunks, feed each chunk to update(), then call final() once.
